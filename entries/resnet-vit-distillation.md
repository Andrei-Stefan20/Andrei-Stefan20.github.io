Vision-Language Models are usually presented as large systems where every component feels fixed. An image enters one side, text comes out of the other, and everything in between is treated almost like a black box.

I wanted to remove one of those components and see what would actually break.

Qwen normally relies on a Vision Transformer to convert images into a sequence of embeddings. My question was whether that encoder was truly indispensable, or whether the language model mainly cared about receiving the right representation at the boundary.

So I tried replacing it with ResNet-50.

Not by retraining the whole multimodal model and not by changing the language model. The idea was to leave Qwen untouched and teach a new visual path to produce something close enough to the embeddings generated by the original ViT.

## The real project was the adapter

![Multi-stage adapter connecting ResNet-50 to Qwen](/images/projects/resnet-vit/adapter2.png)

Replacing one encoder with another takes very little code. Making the new encoder compatible with the rest of the model is where the actual work begins.

A Vision Transformer outputs a flat sequence of visual tokens. ResNet-50 produces hierarchical feature maps at different resolutions and depths. Early layers preserve edges, textures and local detail, while deeper layers carry more abstract semantic information.

The two architectures do not only produce tensors with different shapes. They describe an image in fundamentally different ways.

The adapter became the translator between them.

It receives features from multiple ResNet stages, projects them into compatible spaces, fuses local and semantic information and finally converts the result into the exact sequence expected by Qwen.

The contract is strict. The language model expects a tensor with shape `[B, 196, 2048]`. Producing the same dimensions is necessary, but it is not enough. The representation also needs to preserve the latent structure that Qwen learned during its original multimodal training.

## Distilling a representation, not a label

The original Qwen Vision Transformer acted as the teacher.

Each image was first processed by the native visual encoder and the resulting embeddings were stored. The same image was then passed through ResNet-50, and the adapter attempted to reconstruct the teacher representation from the CNN feature maps.

The initial objective minimized the Mean Squared Error between the original embeddings and the reconstructed ones.

This changes the nature of the problem. The model is not directly learning whether an image contains a tank, an animal or a building. It is learning how Qwen internally represents that image before a single word is generated.

That was the part I found most interesting. A tensor can have the correct shape and still be useless if the information inside it lives in the wrong geometry.

## Four versions instead of one oversized guess

I did not want to build one large adapter immediately and then have no idea which part was actually helping.

The project evolved through four main versions.

The first was a convolutional baseline built around projection and fusion blocks. Its purpose was simple: verify that the mapping was possible at all.

The second added a Transformer Encoder after feature fusion to recover some of the global interaction naturally available in a ViT.

The third focused more on local spatial context and improved the convolutional fusion stage.

The fourth combined both ideas, using stronger local processing together with Transformer-based global reasoning.

This progression was more useful than jumping directly to the most complex model. Each version answered a slightly different question about what the adapter was missing.

## What happens to the tensors

![Detailed adapter architecture and tensor dimensions](/images/projects/resnet-vit/adapter_architecture.png)

ResNet produces several feature maps with different spatial resolutions and channel depths. These maps cannot simply be concatenated and passed forward.

Each stage is first projected into a compatible representation. Higher-resolution features preserve local structure, while deeper maps contribute more abstract information. The fusion path repeatedly combines these signals until they form a single visual representation.

The final blocks reshape that representation into 196 visual tokens, each with dimensionality 2048.

This exact interface kept the experiment focused. I did not redesign Qwen around ResNet. ResNet and the adapter had to fit the model that already existed.

## Training the visual path

The first training stage kept ResNet frozen and optimized only the adapter.

This gave the translation layer a stable input distribution. ResNet continued to produce its pretrained ImageNet features, while the adapter learned how to transform them into the teacher space.

Once the mapping became reasonable, I experimented with joint fine-tuning. ResNet and the adapter were trained together using different learning rates, with a smaller learning rate for the backbone and a larger one for the adapter.

A final adapter-only phase followed the joint training.

The goal was to let the backbone move slightly toward representations that were easier to translate without immediately destroying its useful pretrained features.

## Looking at a real image

![Tank used as a qualitative inference example](/images/projects/resnet-vit/tank.jpg)

This image is useful because it combines several difficult visual elements at once: straight edges, tracks, text, mechanical components and a relatively busy background.

A weak representation might preserve only the broad concept of “vehicle” while losing the local evidence needed for a more precise description. The adapter therefore needs to retain both high-level semantics and enough spatial detail for the language model to use.

The image is not a benchmark by itself. It is a qualitative check that makes certain failures easier to notice.

## The difficult part is not matching the shape

One of the easiest mistakes in this kind of project is treating compatibility as a dimensionality problem.

If Qwen expects `[B, 196, 2048]`, producing `[B, 196, 2048]` may look like success. It is not.

Two tensors can have identical dimensions and represent completely different things. Similar images should occupy meaningful regions, token relationships should remain coherent and the language model should still be able to extract useful information from the sequence.

MSE is a useful starting point because it directly pushes the student representation toward the teacher. It does not guarantee that every important semantic relationship is preserved.

That is where the next experiments become interesting. Cosine objectives could focus more on direction than magnitude. Contrastive alignment could preserve relationships between images. Downstream losses could optimize captioning or visual question answering instead of only the intermediate tensors.

## What I took away from the project

Before working on this, I mostly thought of the Vision Transformer as the component that allowed Qwen to understand images.

After working on it, I started thinking much more about the boundary between the visual and linguistic parts.

The language model never sees pixels. It sees tensors produced by another component.

That does not make the problem easy. Matching dimensions is trivial compared with preserving the latent structure the model has learned to interpret. Still, once the interface is isolated, components that initially appear fixed become open to experimentation.

For me, that is the most valuable result of the project.

It is not simply a CNN replacing a Transformer. It is an attempt to connect two very different representation systems without rebuilding everything around them.
